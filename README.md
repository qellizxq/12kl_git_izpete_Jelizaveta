“Pievienots pirmais projekta fails”
1. Kāda ir atšķirība starp Git un GitHub?

Git ir versiju kontroles sistēma — programma, kas darbojas tavā datorā un pārvalda failu versijas, izmaiņu vēsturi un ļauj atgriezties iepriekšējos stāvokļos.
GitHub ir tiešsaistes platforma, kur Git repozitorijus var uzglabāt mākonī, koplietot ar citiem un sadarboties komandā.

2. Kādas trīs svarīgas komandas vai darbības tika demonstrētas videoklipā?

git add – pievieno izmaiņas sagatavošanas zonai (staging area).
git commit – saglabā izmaiņas versiju vēsturē ar aprakstu.
git push – nosūta commit uz GitHub repozitoriju.

3. Kāpēc ir svarīgi dokumentēt projekta izmaiņas commit ziņojumos?
• Tas palīdz saprast, kas mainīts un kāpēc.
• Atvieglo komandas darbu un sadarbību.
• Palīdz atgriezties pie iepriekšējām versijām, ja rodas kļūdas.
• Nodrošina pārskatāmu projekta attīstības vēsturi.
